# Product

> 範圍、合規、技術架構以 `Splatoon3-武器工具-規格說明書.md`(§2 / §4 / §5)為準;本檔只談策略(為什麼)。衝突時:合規 > 範圍 > 本檔原則。

## Register

product

## Users

Splatoon 3 玩家(休閒到中度)。使用情境是**不定期、低頻**的「想到才查」:臨時想確認某把武器的副/特殊與數值、開打前想隨機抽一把換口味。多半在**手機**上、零碎時間打開,語言橫跨 ja-JP / zh-TW / en。他們要的工作(job-to-be-done):**快速查到武器資料並看懂**,偶爾**抽一把武器**增添開打前的樂趣。非官方、非營利的同好工具,使用者對「免費、可信、不假裝官方」有預期。

## Product Purpose

非官方、非營利的 Splatoon 3 武器查詢與輔助工具。三件事:

1. **武器資料查詢(核心)** — 列表 + 詳情,靜態表格呈現名稱、分類、副/特殊武器、戰鬥數值與裝備效益數值。這是玩家真正回訪的理由。
2. **隨機武器決定器(附加趣味)** — 多重條件篩選後抽一把,無狀態。
3. **11 分類自繪 SVG 美術(視覺亮點)** — 每分類一個代表武器的自繪 SVG,含最低限度動態,嵌入詳情頁與抽選結果。

成功樣貌:玩家覺得「有 Splatoon 味、好逛」,查數值時「清楚、可信」,抽武器時「好玩、想再抽」;同時守住合規(自繪美術、標註來源、標明非官方)。功能優先序由高到低:① 對玩家的實用性與回訪 ② SVG 美術與動態 ③ 開發速度 ④ 前端工程深度。

## Brand Personality

玩家對玩家的口吻:**鮮快(fresh)、自信、有序**。

- **聲音/語氣**:帶街頭塗鴉的活力與一點玩心,但一講到數值就轉為精準、可信賴。像「同好做的、卻做得很專業」的參考工具,而非官方公告腔,也非冷冰冰的企業工具。
- **情緒目標**:列表頁要「逛得開心」、詳情頁要「看得清楚、信得過」、隨機器要「再抽一次」的衝動。
- **視覺主軸(本次定案)**:**折衷路線 —— 以 Inkopolis Splatter(霓虹墨水、噴濺、貼紙字)當品牌骨幹,集中用在列表、隨機器與 SVG 美術;詳情/數值畫面借 Armory Blueprint 的克制與等寬數據紀律,確保可掃描、可讀。**「響亮的品牌、克制的資料」是這個產品的核心張力。

## Anti-references

- **一般 SaaS 儀表板**:藍灰側欄 + 同尺寸卡片網格 + 企業感,毫無 Splatoon 個性。(使用者明確排除)
- **通用 AI 模板**:米色/奶油背景 + 高反差襯線 + 赤陶色,或一堆旋轉卡片堆砌的「一眼看穿是 AI」預設。(使用者明確排除)
- **官方/任天堂外觀**:站名與 UI 不得讓人誤認為官方或任天堂授權工具;不使用任何遊戲 dump 的官方圖檔。(規格 §4.4、§4.3 之合規硬限制,非美學偏好)

## Design Principles

1. **「查得到、看得懂」優先**:回訪核心是資料查詢。任何視覺決定都不得傷害數值的可讀性與掃描效率;資料畫面的對比、對齊、節奏優先於花俏。
2. **響亮的品牌、克制的資料**:把大膽集中在少數招牌時刻(列表氛圍、SVG 美術、抽選揭曉);數值表保持安靜、紀律、等寬對齊。一處放膽,其餘安靜。
3. **自繪即身分**:視覺資產一律自繪 SVG —— 同時是合規策略(避開官方美術著作)與辨識度來源。美術是內容亮點,不是貼上去的裝飾;動態服務於回饋,不喧賓奪主。
4. **誠實的非官方**:明確標示非官方、標註資料來源(splatoon3.ink)、永遠免費。不偽裝成官方,合規本身是設計的一部分。
5. **手機優先、三語平等**:mobile-first;ja-JP / zh-TW / en 三語皆為一等公民,字串不寫死,版面需容納不同語言長度。

## Accessibility & Inclusion

目標等級:**基本**(不設正式 WCAG 等級)。最低要求:語意化標籤、鍵盤可操作、可見的鍵盤 focus。即使不正式宣稱等級,仍維持兩條工程底線:① 內文對比足以閱讀(資料畫面尤其不可用淺灰小字);② 尊重 `prefers-reduced-motion`,所有動態都有降階版本。11 分類的用色不應僅靠顏色區分分類(輔以圖示/標籤),為日後補強色盲友善預留空間。
